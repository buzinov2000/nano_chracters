# Character Pipeline Bot

Телеграм-бот для автоматизации нейтральной части пайплайна разработки персонажей.

---

## Контекст и задача

Рабочий флоу разработки персонажей состоит из двух частей:

**Творческая (остаётся у человека):**
- Отбор скетчей
- Формулировка гипотез и направлений
- Оценка результатов
- Принятие решений

**Нейтральная (автоматизируется ботом):**
- Перевод гипотезы в промпт для генератора
- Отправка промпта в Gemini Image API
- Возврат сетки вариантов с номерами
- Микро-подсказки для следующей итерации
- Итерация промпта по обратной связи

---

## Архитектура

```
[пользователь: файл скетча + текст/голос]
        ↓
[Telegram Bot — курьер, своей обработки нет]
        ↓
[Gemini Flash — промпт-агент]
  получает: скетч + гипотеза пользователя
  отдаёт: промпт для Nano Banana + 2-3 подсказки
        ↓
[Gemini Image API (Nano Banana 2) — генерация 4–8 вариантов]
        ↓
[бот возвращает превью сеткой с номерами]
[бот возвращает подсказки для следующей итерации]
        ↓
[пользователь: "3 6" → полное разрешение]
[пользователь: правка текстом → новая итерация]
```

**Арт-директор — отдельно, не в боте:**
Живёт в Claude Projects с брифами проекта в файлах. Вызывается вручную когда нужна
экспертная оценка. Уже оплачен в рамках подписки Claude.

---

## Агенты

### Agent 1 — Промпт-агент (Gemini Flash)
- **Вход:** картинка скетча + описание гипотезы человеческим языком
- **Задача:** перевести в развёрнутый промпт для Nano Banana
- **Дополнительно:** после промпта предлагает 2–3 микро-подсказки для следующей итерации
  (например: "Попробовать сетку 2×2 с разными материалами?", "Сделать версию в стиле мягкой игрушки?")
- **Контекст:** кастомная инструкция под стиль студии (3D, материалы, объём)
- **Модель:** Gemini 2.5 Flash (дешевле Haiku, нативно мультимодален, один провайдер)

### Арт-директор — вне бота
- **Где:** отдельный Project в Claude (уже оплачен)
- **Как:** бриф и референсы хранятся в файлах проекта
- **Вызов:** вручную, когда нужна экспертная оценка вариантов
- **Модель:** Claude Sonnet 4.6 (через существующую подписку)

## Интерфейс бота — команды

| Команда | Действие |
|--------|----------|
| [файл + текст/голос] | запуск нового цикла генерации |
| `[номера через пробел]` | вернуть выбранные файлы в полном разрешении |
| `/промпт` | показать текущий промпт текстом |
| `/промпт [правки]` | обновить промпт и перегенерировать |
| `/старт` | начать новый проект, сбросить контекст |

*Голосовые сообщения: Telegram передаёт как аудиофайл — транскрипция через Gemini
(у него есть аудио-понимание, ключ уже есть). Добавить на позднем этапе.*

## Стек

| Компонент | Решение |
|-----------|---------|
| Бот | Python, библиотека `python-telegram-bot` |
| Промпт-агент | Google AI API, Gemini 2.5 Flash |
| Генерация картинок | Google AI API, Gemini Image (Nano Banana 2) |

| Хостинг | существующий VPS, отдельный systemd-сервис рядом с текущим ботом |

**Один API-ключ для всего бота** — Google AI Studio.

## Примерные расходы

| Статья | Цена |
|--------|------|
| Gemini Flash (промпт-агент) | ~$0.0005 за итерацию |
| Nano Banana 2 (1K, 8 картинок) | ~$0.36 за итерацию |
| **10 итераций в день** | **~$3.6/день при активной работе** |
| **Месяц активной работы** | **~$80–100/мес** |

**Что уже оплачено и используется бесплатно для бота:**
- Claude подписка $20/мес → арт-директор в Projects
- Freepik $39/мес → веб-интерфейс для ручной работы (API отдельный, не используем)
- ChatGPT $12/мес → можно отключить, функции перекрыты

**Важно:** бесплатный тир Gemini Image (500–1000 картинок/день) работает только через
веб-интерфейс AI Studio, не через API. Для бота нужен платный Google Cloud аккаунт с billing.

## Этапы реализации

- [ ] **Этап 1** — скелет бота: принимает файл + текст, отвечает эхом
- [ ] **Этап 2** — промпт-агент: Gemini Flash генерирует промпт из скетча и описания + 2-3 подсказки
- [ ] **Этап 3** — генерация: Nano Banana 2 возвращает превью сеткой с номерами
- [ ] **Этап 4** — выбор файлов по номеру в полном разрешении
- [ ] **Этап 5** — итерация промпта через текстовые правки (`/промпт [правки]`)
- [ ] **Этап 6** — деплой на VPS рядом с существующим ботом (systemd-сервис)
- [ ] **Этап 7** *(опционально)* — транскрипция голосовых через Gemini Audio

## Заметки

- Бот не хранит картинки — только передаёт потоком через API
- Нагрузка на VPS минимальная, вся обработка на стороне Google AI API
- Голосовые сообщения Telegram передаёт как аудиофайл — транскрипция отдельным шагом
- Состояние сессии (текущий промпт, последние картинки) живёт в памяти процесса —
  при перезапуске сбрасывается (на первом этапе нормально)
- Один провайдер (Google AI) = один ключ = меньше сложности
