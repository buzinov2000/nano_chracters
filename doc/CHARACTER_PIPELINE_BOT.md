# Character Pipeline Bot

Телеграм-бот для автоматизации нейтральной части пайплайна разработки персонажей.

---

## Контекст и задача

Рабочий флоу разработки персонажей состоит из двух частей:

**Творческая (остаётся у человека):**
- Отбор скетчей
- Формулировка гипотез и направлений
- Оценка результатов
- Принятие решений

**Нейтральная (автоматизируется ботом):**
- Перевод гипотезы в промпт для генератора
- Отправка промпта в Gemini Image API
- Возврат сетки вариантов с номерами
- Микро-подсказки для следующей итерации
- Итерация промпта по обратной связи

---

## Архитектура

```
[пользователь: скетч + рефы (опционально) + текст]
        ↓
[Telegram Bot — курьер, своей обработки нет]
        ↓
[Gemini Flash — промпт-агент]
  получает: скетч + рефы + гипотеза пользователя
  отдаёт: промпт для генератора + 2-3 подсказки
        ↓
[Gemini Image API — генерация 4 вариантов параллельно]
        ↓
[бот возвращает превью сеткой с номерами]
[бот возвращает подсказки для следующей итерации]
        ↓
[пользователь: "1 3" → полное разрешение]
[пользователь: /промпт правки → новая итерация]
```

**Арт-директор — отдельно, не в боте:**
Живёт в Claude Projects с брифами проекта в файлах. Вызывается вручную когда нужна
экспертная оценка. Уже оплачен в рамках подписки Claude.

## Агенты

### Agent 1 — Промпт-агент (Gemini Flash)
- **Вход:** картинка скетча + референсные изображения (опционально) + описание гипотезы человеческим языком
- **Задача:** перевести в развёрнутый промпт для генератора
- **Дополнительно:** после промпта предлагает 2–3 микро-подсказки для следующей итерации
  (например: "Попробовать сетку 2×2 с разными материалами?", "Сделать версию в стиле мягкой игрушки?")
- **Контекст:** кастомная инструкция под стиль студии (3D, материалы, объём)
- **Модель:** Gemini 2.5 Flash (дешевле Haiku, нативно мультимодален, один провайдер)

### Арт-директор — вне бота
- **Где:** отдельный Project в Claude (уже оплачен)
- **Как:** бриф и референсы хранятся в файлах проекта
- **Вызов:** вручную, когда нужна экспертная оценка вариантов
- **Модель:** Claude Sonnet 4.6 (через существующую подписку)

## Интерфейс бота — команды

| Команда | Действие |
|--------|----------|
| [скетч + текст] | запуск нового цикла генерации |
| [несколько фото + текст] | первое фото = скетч, остальные = рефы |
| `[номера через пробел]` | вернуть выбранные файлы в полном разрешении |
| `/промпт` | показать текущий промпт текстом |
| `/промпт [правки]` | обновить промпт и перегенерировать |
| `/старт` | начать новый проект, сбросить контекст |

> **Особенность:** кириллические команды (`/старт`, `/промпт`) обрабатываются через текстовый хэндлер, т.к. Telegram Bot API не поддерживает не-латинские команды в `CommandHandler`.

## Стек

| Компонент | Решение |
|-----------|---------|
| Бот | Python 3.11+, `python-telegram-bot` v21+ (async) |
| SDK | `google-genai` (не `google-generativeai`) |
| Промпт-агент | Gemini 2.5 Flash |
| Генерация картинок | Gemini 2.5 Flash Image через `generate_content` |
| Сборка сетки | Pillow (пропорции сохраняются) |
| Конфиг | `python-dotenv` |
| Арт-директор | Claude Projects (вне бота) |
| Хостинг | VPS, systemd-сервис |

**Один API-ключ для всего бота** — Google AI Studio.

## Структура проекта

```
nano_characters/
├── doc/
│   ├── CHARACTER_PIPELINE_BOT.md   ← этот файл
│   └── DEVELOPMENT_PLAN.md         ← план разработки
├── prompts/
│   └── prompt_agent.txt        ← системный промпт для агента
├── bot.py                      ← точка входа, хэндлеры Telegram
├── agent.py                    ← промпт-агент (Gemini Flash)
├── imagen.py                   ← генерация картинок (Gemini Image)
├── grid.py                     ← сборка сетки превью с номерами (Pillow)
├── session.py                  ← состояние сессии (in-memory)
├── config.py                   ← загрузка .env, константы
├── .env                        ← API ключи (в .gitignore)
├── .gitignore
└── requirements.txt
```

## Реализованные этапы

- [x] **Этап 0** — инициализация проекта, структура, зависимости
- [x] **Этап 1** — скелет бота: принимает файл + текст, отвечает эхом
- [x] **Этап 2** — модуль сессий (in-memory)
- [x] **Этап 3** — промпт-агент: Gemini Flash генерирует промпт + подсказки
- [x] **Этап 4** — генерация картинок: 4 параллельных вызова Gemini Image
- [x] **Этап 5** — сборка сетки превью с номерами (Pillow)
- [x] **Этап 6** — полный цикл: скетч → промпт → картинки → сетка → выбор
- [x] **Этап 7** — итерация промпта через `/промпт [правки]`
- [x] **Этап 8** — полировка: retry, typing indicator, таймер, error handler
- [x] **Доп.** — поддержка медиагрупп (несколько фото: скетч + рефы)

## Не реализовано (бэклог)

- Транскрипция голосовых через Gemini Audio
- Деплой на VPS (systemd)
- Персистентность сессий между перезапусками
- A/B тест качества промптов Gemini Flash vs Claude Haiku

## Заметки

- Бот хранит картинки в памяти процесса (в сессии) — при перезапуске сбрасывается
- Нагрузка на VPS минимальная, вся обработка на стороне Google AI API
- Gemini Image возвращает одну картинку за вызов — для N вариантов делается N параллельных async-вызовов
- Один провайдер (Google AI) = один ключ = меньше сложности
